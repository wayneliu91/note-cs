# 链表

## 题型说明

```wiki
很常考，是【重中之重】

题型有限，代码【实现难】
这类题⽬考察的不是算法，不像动态规划，题型很多，他纯粹考察的是候选⼈的【编程能⼒】。所以，⾯试中基本上都是【原题】或者在原题上【稍加改造】。

如何准备这类题型？
虽然代码实现难，很容易写出bug，但这类题⽬不难准备⾯试。
只要把这节课布置的13道题都写熟练，基本上链表的题⽬就没有问题了。
```

## 解题技巧

```wiki
链表相关的问题都会涉及“遍历”，核⼼是通过“画图举例”确定遍历的“三要素”： 
1. 遍历的结束条件：p==null or p.next==null …
2. 指针的初始值：p=head or …
3. 遍历的核⼼逻辑：…（视题⽬要求⽽定）
特殊情况处理：是否需要对头节点、尾节点、空链表等做特殊处理？
引⼊虚拟节点：是否可以通过添加虚拟节点简化编程？
```
